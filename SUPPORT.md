# Support

Thanks for using this project! Here's how to get help and where to report issues.

## 📚 Read the Docs First

Before reaching out, please check:

- The [README](./README.md) for an overview and quickstart guide
- The [documentation](https://open-sesam.github.io/sesam/) for full API references and guides
- Existing [GitHub Discussions](../../discussions) and [Issues](../../issues) — your question may already be answered

---

## 💬 Asking Questions

For general questions, usage help, and open-ended discussion, please use **[GitHub Discussions](../../discussions)**.

**Good topics for Discussions:**
- "How do I do X?"
- "What's the recommended pattern for Y?"
- "Has anyone integrated this with Z?"
- Ideas, feature requests, and feedback

Please search existing discussions before opening a new one.

---

## 🐛 Reporting Bugs

Found something broken? Please [open a GitHub Issue](../../issues/new/choose).

**Before filing a bug report:**
1. Make sure you're on the latest version
2. Check that the issue hasn't already been reported
3. Reproduce the problem with a minimal example

**A good bug report includes:**
- A clear, descriptive title
- Steps to reproduce the issue
- What you expected to happen vs. what actually happened
- Your environment (OS, language version, library version)
- Any relevant logs, stack traces, or screenshots

---

## ✨ Requesting Features

Feature requests are welcome! Please [open a GitHub Issue](../../issues/new/choose) using the feature request template, or start a conversation in [GitHub Discussions](../../discussions).

Describe the problem you're trying to solve, not just the solution you have in mind — this helps us understand the use case and find the best approach.

---

## 🔒 Security Vulnerabilities

**Please do not report security vulnerabilities through public GitHub Issues.**

If you discover a security issue, please disclose it responsibly according to our [security policy](./SECURITY.md). We'll acknowledge your report within 48 hours and work with you on a fix and disclosure timeline.

---

## 🤝 Contributing

Want to fix a bug or add a feature yourself? We'd love your help! Take a look at our [contribution guidelines](./CONTRIBUTING.md) to get started.

---

## ⏱️ Response Times

This is a community-supported open source project maintained by volunteers. We aim to respond to issues and discussions within a few days, but response times may vary. There is no guaranteed SLA.

If a thread has gone quiet and you're still stuck, a polite bump is welcome.
