# Security Policy

## Supported Versions

Security fixes are done as new versions. Right now we don't backport fixes for old version.
We advise you to keep your versions up-to-date (check with `sesam doctor`).

## Reporting a Vulnerability

Please report security issues via the [advisory board](https://github.com/open-sesam/sesam/security/advisories)
and NOT as public issue. This is only available to the developers and the reporter and therefore gives us time to
verify your report and potentially ship a fix before it is used

### Best report practices

- Please ensure the issue is reproducible on main.
- Ideally a fully working, end-to-end reproducer is provided.
- Please ensure the reproducer is real-world and not simulated or abstracted.
- Please ensure the reproducer demonstrably violates a security boundary.
